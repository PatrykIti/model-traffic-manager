# Official Documentation

This directory is the official documentation space for the application itself.

## Why `docs/` exists at the repository root

`docs/` is reserved for the product/application story:

- what the router is
- how its runtime behavior works
- how operators and contributors should understand the system
- how the public-facing logic, configuration, and operations model fit together

This is intentionally separate from `_docs/`, which is the internal delivery space used for planning, task tracking, changelog history, and AI-assisted development workflow.
